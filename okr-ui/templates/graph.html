Cross-team dependencies

Generated by /dependency-graph. Click a node to jump to that KR.

!
This graph only shows dependencies that someone wrote down.

Edges come from each KR's depends_on field — the declared coupling between teams. The most damaging dependencies are usually the ones nobody bothered to write down: that legal will turn the customer contract in 5 days, that the hardware vendor will ship on time, that a key person won't quit before the launch. The graph cannot see those.

Use this as a starting point for the conversation, not as the map. Each team page now has an Unwritten risks field where you should be naming the things this graph misses.
